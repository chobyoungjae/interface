# 불량체크일지 PRD 생성을 위한 메타 프롬프트

## 목적
이 메타 프롬프트는 AI가 불량체크일지 시스템의 PRD를 정확하게 생성하도록 안내합니다.

---

## 프롬프트

```
당신은 제조업 생산관리 시스템 전문가입니다.
아래 요구사항을 바탕으로 "불량체크일지" 웹 입력 시스템의 PRD를 작성하세요.

## 시스템 개요
- 제조 공정에서 발생하는 포장지/박스 불량을 기록하는 웹 애플리케이션
- 기술 스택: Next.js 15 + TypeScript + Tailwind CSS + Google Sheets API

## 데이터 소스

### 1. BOM 스프레드시트 (제품/원자재 정보)
- ID: 1WlabNwKwMWW-cKA_go2Q680g3S7wYDpb5iCzE_YIZdM
- 시트2: 생산품 목록 (A열: 코드, B열: 품목명, F열: 소모품코드, G열: 소모품명)
- 시리얼로트 시트: 로트 정보 (A열: 품목코드, B열: 제품명, D열: 로트번호, F열: 재고수량)
  - 헤더: 2행

### 2. 저장 스프레드시트 (데이터 저장/참조)
- ID: 10zoJnnRuQQDh_VYR5QdVPjjH_Wg4VEcr9t_E-ytp0iQ
- 시트1: 불량 데이터 저장 위치
- B시트: 작성자 목록 (B열: 이름, C열: 부서 - "생산팀" 필터)
  - 헤더: 4행

## 입력 필드 명세

### 1. 작업자 (드롭다운)
- 소스: 저장시트 > B시트
- 조건: C열 = "생산팀"
- 표시: B열 (이름)

### 2. 라인 (드롭다운 - 고정값)
- 옵션: 1라인, 2라인, 3라인, 4라인, 수작업, 배합실

### 3. 생산품 선택 (드롭다운)
- 소스: BOM시트 > 시트2
- 표시: A열(코드) / B열(품목명)
- 중복 제거 필수

### 4. 포장지 자동선택
- 트리거: 생산품 선택 시
- 로직: BOM 시트2에서 A열(생산품목코드)이 선택된 제품과 동일한 모든 행 중 F열이 6으로 시작하는 6자리 코드
- 표시: F열(코드) / G열(품목명)

### 5. 포장지 로트 선택 (드롭다운)
- 트리거: 생산품 선택 → 포장지 자동확정 → 로트 드롭다운 활성화
- 소스: BOM시트 > 시리얼로트 시트
- 매칭: 4번 포장지의 F열(코드) = 시리얼로트 시트의 A열(품목코드)
- 표시: D열(로트번호) / F열(재고수량)
- 주의: 동일한 포장지 코드에 여러 로트가 존재할 수 있음 → 매칭되는 모든 로트를 드롭다운에 표시

### 6. 포장지 불량 수량 (숫자 입력 x 4)
- 실링불량, 중량불량, 날인불량, 자체불량

### 7. 박스 자동선택
- 트리거: 생산품 선택 시
- 로직: BOM 시트2에서 A열(생산품목코드)이 선택된 제품과 동일한 모든 행 중 F열이 7로 시작하는 6자리 코드
- 표시: F열(코드) / G열(품목명)

### 8. 박스 불량 수량 (숫자 입력 x 4)
- 박스오염, 파손, 날인불량, 기타

### 9. 특이사항 (텍스트 입력 x 3)
- 내용, 개선조치사항, 완료여부

## 저장 형식 (시트1, 가로 방향)
A: 타임스탬프 | B: 작업자 | C: 라인 | D: 생산품코드 | E: 생산품명 |
F: 포장지코드 | G: 포장지명 | H: 포장지로트 | I~L: 포장지불량(4개) |
M: 박스코드 | N: 박스명 | O~R: 박스불량(4개) | S~U: 특이사항(3개)

## PRD에 포함할 내용
1. 프로젝트 개요 및 목적
2. 기술 스택
3. 데이터 소스 구조 (시트별 열 매핑)
4. 입력 필드 상세 명세 (트리거, 로직, 표시 형식)
5. 저장 데이터 구조 (열별 헤더 및 내용)
6. 화면 흐름도 (ASCII 또는 다이어그램)
7. API 엔드포인트 설계
8. 환경 변수 목록

## 코드 식별 규칙
- 포장지: F열이 6으로 시작하는 6자리 숫자
- 박스: F열이 7로 시작하는 6자리 숫자

위 요구사항을 바탕으로 개발팀이 바로 구현할 수 있는 상세한 PRD를 작성하세요.
```
